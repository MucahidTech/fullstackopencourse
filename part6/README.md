# Part 6 - Advanced State Management

Exploring modern state management solutions in React: Zustand for client-state, TanStack Query for server-state, and Context API for cross-cutting concerns.

## 📁 Projects

| Project                | Description                                                                                                  |
| :--------------------- | :----------------------------------------------------------------------------------------------------------- |
| **unicafe-zustand**    | Feedback app reimplemented with Zustand (exercise 6.1)                                                       |
| **anecdotes-zustand**  | Full-featured anecdote app with Zustand, JSON Server, filtering, and testing (exercises 6.2 - 6.15)          |
| **anecdotes-tanstack** | Anecdote app using TanStack Query for server-state and Context API for notifications (exercises 6.16 - 6.22) |

## 🧠 Core Techniques

- 🗄️ **Zustand** – Client-state management with `actions` pattern and custom hooks
- ⚛️ **TanStack Query** – Server-state management with `useQuery`, `useMutation`, and cache invalidation
- 📦 **Context API** – Cross-cutting state (notifications) without prop drilling
- 🌐 **Fetch API** – Native HTTP requests for server communication
- 🧪 **Testing** – Testing Zustand stores and custom hooks with Vitest and React Testing Library
- 🧩 **Custom Hooks** – Encapsulating state logic for reusable and clean components
