# Part 2 - Communicating with Server

Building React apps that handle forms, fetch data from backend, and manage side effects.

## 📁 Projects

| Project        | Description                                                                                                                             |
| :------------- | :-------------------------------------------------------------------------------------------------------------------------------------- |
| **courseinfo** | Dynamic course rendering using `.map()` and modular components (exercises 2.1-2.5)                                                      |
| **phonebook**  | Full CRUD contact app with search, add/delete/update, backend integration, and notifications (exercises 2.6-2.17)                       |
| **countries**  | Fetches country data from REST API; includes search filter, country details, and weather info from OpenWeatherMap (exercises 2.18-2.20) |

## 🧠 Core Techniques

- 🗂️ **Rendering collections** – Using `.map()` with unique `key` props
- 📝 **Controlled forms** – Managing form input state with `useState`
- 🌐 **Data fetching** – Using `axios` and `useEffect` to get data from server
- 🔄 **HTTP requests** – GET, POST, PUT, DELETE operations
- 🧩 **Component composition** – Building reusable UI pieces
- 🧹 **Service modules** – Extracting API calls into separate modules
- ⚙️ **Environment variables** – Securing API keys with Vite's `.env`
- 💅 **Basic styling** – CSS rules and inline styles for notifications
